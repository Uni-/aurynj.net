언어

자연어보다 인공어를 더 많이 하게 된 흔한 CSE guy입니다.

자연어

  • 한국어 모어 화자입니다.
  • 영어는 뉴스나 기술 문서, 논문은 읽지만 면접을 볼 정도는 아닙니다.
  • 일본어의 일부 낱말과 활용을 구사할 수 있습니다.

인공어

주력 언어로는 C, C++, Java를 사용합니다. C11, C++11, JDK 6 미만에 호환되는 코드는 웬만하면 작성하지 않습니다. 스스로의 코딩 컨벤션에 엄격합니다.

sh/bash 스크립트, cmd batch 스크립트를 작성할 수 있습니다. 그러나 개인 목적의 스크립팅에는 ECMAScript를 사용합니다. 유닉스 계열에는 Node.js, 윈도우에는 WSH가 있으며, 양쪽 모두에서 사용할 수 있는 모질라 js 바이너리도 있습니다.

그 밖에 한때 열심히 써 봤던 언어로는 Visual Basic, C#, Python, Haskell, OCaml, Scheme, Go, Coq Gallina, Rust, ……. 다른 언어는 거의 전혀 모릅니다.

소프트웨어

여기서 소프트웨어는 프로그래밍 가능한 하드웨어 상에서 복제 가능한 프로그램을 의미합니다.

운영체제

Windows와 Linux를 사용합니다.
  • Windows라는 이름의 여러 운영체제를 사용합니다. Internet Explorer 호환성 실험과, 물리 메모리 보호를 어기는 옛날 개발 도구들 때문에 Windows 98을 쓸 때가 있었습니다. 같은 목적으로 Windows XP SP1을 쓰기도 합니다. XP SP2/SP3을 써도 될 때는 Windows 7의 Windows XP Mode를 사용합니다. Windows Vista가 나왔을 때 일찍이 NT5 계열(Windows XP)에서 NT6 계열(Windows Vista)로 갈아탄 바 있습니다. 개인 노트북에는 Windows 8.1이 설치되어 있으며, 현 직장에서는 몇몇 이유로 Windows 7을 쓰고 있습니다.
  • Linux 커널 기반의 운영체제는 그다지 다양한 버전으로 사용하고 있지 않습니다. 상대적으로 이쪽의 애플리케이션들은 (1) F/OSS로, 최신 플랫폼에 맞추어 꾸준히 업데이트되고 있거나, (2) 운영체제가 제공하는 패키지 관리자를 이용해 호환성이 맞는 환경을 상대적으로 쉽게 조성할 수 있기 때문입니다. 항상 최신 버전의 Fedora를 사용합니다. 너무 쉬운 배포에 익숙해지지는 않기 위해, Ubuntu 계열은 의도적으로 피하는 편입니다.
사용하고 있는 스마트폰은 Android 기반으로, 현재 Android 5.0 Lollipop 계열 펌웨어로의 업그레이드를 제공합니다. OS X이나 iOS 계열은 사용하지 않고 있습니다.

하드웨어

시스템

2009년 이래로, 개인용으로는 데스크탑을 잘 쓰지 않습니다. 대량의 컴퓨팅과 데이터에 어디서든 접근하는 데 가장 좋은 방법은, 클라우드 솔루션과 회선을 구매하는 것이 아니라 자기 자신이 움직이는 클라우드가 되는 것입니다. 아직 클라우드와 터미널 간의 동기화는 프로세스 대수 이론에서만큼이나 성가시기 그지없습니다.

노트북은 주로 HP 제품을 구매합니다. 무게와 디자인에 크게 신경쓰지 않고, 그 외의 성능과 내구성이 제품을 고르는 이유가 됩니다. 2013년까지는 Compaq 430을 사용했으며, 지금은 Pavillion 14-b103tx를 사용합니다.

모바일

2012년 중순 17만원 대란 때 구매한 삼성 Galaxy S3를 아직 사용하고 있습니다. 굉장히 운이 좋았는데, 그때 기기를 두 대 사서 하나는 가족이 쓰고 있습니다.

2013년에는 개발용 기기로 Galaxy Note 10.1을 구매했습니다. 아무래도 애플리케이션 개발은 UI 싸움이라 대형 화면 개발에 실제 기기가 도움이 많이 됐습니다. Nexus 10을 사고 싶었지만 없어서 못 샀습니다.

키보드

키보드를 공들여 구매하지 않습니다. 덕분에 운동을 하지 않는 몸이 더욱 빨리 망가지고 있습니다. 그나마 노트북의 펜타그래프 키보드는 싸구려 멤브레인 키보드보다 괜찮습니다.

지금은 어쩌다 마주친 IBM Model F AT 키보드를 사용합니다. 키보드를 특별히 구매할 계획이나 생각은 여전히 없습니다.
